### 1.4 WaveConfig (272 штуки)

Путь: `Assets/Game/Configs/Waves/`  
Имя: `Level01_Wave01.asset` … `LevelXX_WaveYY.asset`

Поля каждого ассета:
```
SubWaves[]
  - Enemy: <ссылка на EnemyConfig>
  - Count: int
  - Interval: float
  - PathIndex: int
DelayAfter: float
Reward: int
```

#### Сводная таблица по всем уровням

**Нотация subwave:** `[тип×N@Xs pP]` → Enemy=ufo-тип, Count=N, Interval=X, PathIndex=P  
Несколько subwave через `·` — последовательный спавн внутри одной волны.

## Фаза 2 — Префабы уровней

Путь: `Assets/Game/Content/Levels/`  
Имя: `Level01.prefab` … `Level25.prefab`

Каждый префаб содержит:
- **Path компоненты** — массив Waypoint-ов, количество соответствует PathIndex
- **TowerSlot** — количество слотов по таблице (3–8 штук)
- **PlayerBase** — объект с компонентом базы
- **SpawnPoint** — точка старта врагов (по одному на каждый путь)
- **Тайловая карта** — визуальный дизайн пути из Kenney-тайлов

### Типы компоновок по уровням

| Группа уровней | Путей | Слотов | Пример |
|----------------|-------|--------|--------|
| L1–3 | 1 | 3 | Прямой / с поворотами |
| L4–5 | 2 | 4 | Split-путь |
| L6–7 | 1 | 4 | Длинный / лабиринт |
| L8–9 | 2 | 5 | Split расширенный |
| L10–12 | 2 | 5 | Аналогично L8–9 |
| L13–15 | 2 | 6 | Dense layout |
| L16–18 | 2 | 6–7 | + 7 слотов с L18 |
| L19 | 3 | 7 | Первый triple path |
| L20 | 2 | 7 | — |
| L21 | 2 | 7 | — |
| L22 | 2 | 8 | — |
| L23 | 3 | 8 | Triple path |
| L24–25 | 3 | 8 | — |

> **Повторное использование префабов:** уровни внутри одного тира со схожей конфигурацией (тот же тип пути, то же кол-во слотов) могут использовать один и тот же базовый префаб с разными `LevelConfig`. Минимальный уникальный набор — ~10 базовых layout-ов.
