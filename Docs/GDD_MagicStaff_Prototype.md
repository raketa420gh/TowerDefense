# GDD — Magic Staff Roguelike (Prototype v0.1)

> Платформа: Mobile (Android/iOS)  
> Жанр: Roguelike / Action  
> Дата: 2026-04-16  

---

## 1. КОНЦЕПЦИЯ

Игрок — маг, спускающийся в бесконечное подземелье. Главное оружие — **магический посох**, собранный из модульных деталей. Каждый забег уникален благодаря случайным усилениям и сочетанию деталей посоха. Игрок умирает, возвращается в меню, немного прокачивается и спускается снова.

### Уникальная механика
Посох — не просто статат. Это конструктор из слотов, каждый из которых влияет на поведение, визуал и эффекты оружия.

## 2. СТРУКТУРА ПОСОХА

```
[ АРТЕФАКТ ]       ← вершина, активный элемент (меняет тип урона/эффекта)
[ НАВЕРШИЕ ]       ← оправа артефакта (усиливает/модифицирует эффект)
[ РУКОЯТЬ ]        ← хват (влияет на скорость атаки, восстановление маны)
[ ДРЕВКО ]         ← вид дерева (влияет на радиус, пассивные бонусы)
[ НАВЕРШИЕ СНИЗУ ] ← украшение (косметика или небольшой бонус)
```

### Слоты и примеры вариантов

| Слот | Варианты (прототип) | Эффект |
|---|---|---|
| Артефакт | Огненный кристалл, Ледяная сфера, Молния-рун | Тип урона + спецэффект |
| Навершие | Золотое, Серебряное, Обсидиановое | +урон / +скорость / +AOE |
| Рукоять | Кожаная, Деревянная, Металлическая | Cooldown / Mana regen |
| Древко | Дуб, Ясень, Эбен | +радиус / +пробивание / +отдача |
| Нижнее навершие | Стальной шип, Хрустальный шар | Пассивный бонус или косметика |

## 3. ЭКРАНЫ (FLOW)

```
[MAIN MENU]
    ├── [НАСТРОЙКА ПОСОХА] ← модификация слотов (разблокированные детали)
    └── [В ПОДЗЕМЕЛЬЕ]     ← старт забега

[GAMEPLAY — УРОВЕНЬ]
    └── Бесконечные волны врагов →
        └── Каждые N волн: [ЭКРАН ВЫБОРА УСИЛЕНИЯ (3 из 10)]
            └── Игрок умирает →
                [ЭКРАН СМЕРТИ / РЕЗУЛЬТАТЫ]
                    └── Небольшой постоянный прогресс (валюта/опыт)
                        └── → [MAIN MENU]
```

## 4. ГЕЙМПЛЕЙ — УРОВЕНЬ (ПРОТОТИП)

- **Арена:** замкнутая локация (подземелье, один зал)
- **Управление:** виртуальный джойстик (движение) + автоатака по ближайшему врагу  
  (вариант: tap-to-aim, решается в плейтесте)
- **Волны:** бесконечные, нарастают по количеству и скорости каждые 30–60 сек
- **Смерть:** HP = 0 → конец забега

### Автоатака посоха
- Посох стреляет автоматически в ближайшего врага
- Тип снаряда/эффекта = зависит от артефакта
- Cooldown = зависит от рукояти

## 6. ПУЛ УСИЛЕНИЙ (10 штук для прототипа)

Каждые N волн игроку предлагается выбрать **1 из 3** случайных:

| # | Название | Эффект |
|---|---|---|
| 1 | Огненный след | Снаряды оставляют горящую зону |
| 2 | Двойной выстрел | Каждая атака выпускает 2 снаряда |
| 3 | Заморозка | 20% шанс заморозить врага на 1 сек |
| 4 | Цепная молния | Снаряд рикошетит на 2 врагов |
| 5 | Притяжение | Снаряды слабо притягивают врагов |
| 6 | Щит маны | Раз в 10 сек поглощает 1 удар |
| 7 | Ускорение | +15% скорость атаки |
| 8 | Пробивание | Снаряды пронизывают врагов насквозь |
| 9 | Взрыв при смерти | Убитый враг взрывается, нанося AOE |
| 10 | Восстановление | Каждые 5 убийств +5 HP |

## 7. МЕТА-ПРОГРЕССИЯ (POST-DEATH)

После смерти игрок получает **Магические Осколки** (валюта) = f(волны, убийства).

### Постоянные улучшения (мета-дерево, прототип: 3 узла)

| Узел | Эффект |
|---|---|
| Начальное HP +10 | Старт с большим запасом здоровья |
| Стартовое усиление | Начать забег с 1 случайным бонусом |
| Скорость разблокировки | Новые детали посоха дешевле |

## 9. DI-АРХИТЕКТУРА (Zenject)

### Контексты

- **ProjectContext** — синглтоны, живущие между сценами: `MetaProgressionService`, `StaffDefinition`
- **SceneContext (Gameplay)** — всё игровое: `WaveManager`, `UpgradeManager`, `PlayerController`, `EnemyRegistry`, `ProjectilePool`, `EnemyPool`
- **SceneContext (MainMenu)** — `StaffModificationController`

### Инсталлеры

| Installer | Что биндит |
|---|---|
| `ProjectInstaller` | `MetaProgressionService` (FromNewScriptableObjectResource), `StaffDefinition` (AsSingle) |
| `GameplayInstaller` | `WaveManager`, `UpgradeManager`, `PlayerController`, `EnemyRegistry`, `ProjectilePool`, `EnemyPool` |
| `MainMenuInstaller` | `StaffModificationController` |

### Правила

- Никаких `Instance` / `FindObjectOfType` — только `[Inject]` или конструктор
- Фабрики снарядов и врагов через `IFactory<>` (Zenject PlaceholderFactory)
- `GameManager` не нужен как синглтон — его ответственность распределяется между `WaveManager` (волны) и `GameFlowController` (переходы между состояниями), оба инжектируются там, где нужны

## 10. КЛЮЧЕВЫЕ РИСКИ ПРОТОТИПА

| Риск | Митигация |
|---|---|
| Автоатака скучная | Плейтест: добавить лёгкий aim-assist / tap-to-fire |
| Волны однообразные | Ввести мини-боссов раньше, чем покажется нужным |
| Усиления не ощущаются | Яркий визуал + звук при выборе |
| Посох не читается визуально | Модульный спрайт с чёткими слотами |
